Professional C++

Notes taken from a really good book

Posted by Max on November 22nd, 2019 Programming

I have been reading a basic introduction book to C++, called Professional C++ by Nicholas A. Solter and Scott J. Kleper. While I am only at Chapter 7, I feel the content of Chapter 1 already gives a really good overview of C++ and some suggestions of the low-level techniques. Therefore, I decided to post some of my personal notes here. And I encourage you to read the book if you want to get more detailed content.

Strongly-typed Enum

  • enum will convert each class to integer
  • enum class will not, and thus recommended
    e.g.
    enum class PieceType
    {
      King = 1,
      Queen,
      Rook = 10,
      Pawn
    };

Array

  • Try to avoid C style array as much as possible
    e.g.
    int myArray[3];
    int myArray[] = {1,2,3};
  • use std::array
    e.g.
    array<int, 3> arr = {9, 8, 7};
    arr.size(); // for array size
    arr[1]; // for array value
    • Note both std::array and C style array have a fixed size
  • use std::vector
    e.g.
    vector<int> myVector = { 11, 22 };
    myVector.push_back(1); // add element
    myVector[0]; // access element
  • Structured bindings
    • must use auto
      e.g.
    // array
    std::array<int, 3> values = { 11, 22, 33 };
    auto [x, y, z] = values; // declare variables
     
    // struct
    struct Point { double mX, mY, mZ; };
    Point point;
    point.mX = 1.0; point.mY = 2.0; point.mZ = 3.0;
    auto [x, y, z] = point;

Loop

  • Range-based for loop
    • allow iterating elements with a copy
      e.g
      std::array<int, 4> arr = {1, 2, 3, 4};
      for (int i : arr) {
        std::cout << i << std::endl; // note each element is copied to i, not reference!
      }
  • Initializer list
    • allow function to have a variable number of arguments
      e.g
      #include <initializer_list>
       
      int makeSum(initializer_list<int> lst)
      {
        int total = 0;
        for (int value : lst) {
          total += value;
        }
        return total;
      }

Pointer and Dynamic Memory

Stack

  • each function is a stack frame
  • stack frame lays on top of each other based on execution sequence (latest on the top)
  • provides isolated memory
  • when function is done, the stack frame is gone, with the variables declared inside, so no dealloaction needed
  • variables passed from one function to the other are copied

Heap

  • heap is independant of any stack frames or functions
  • it exists even if the function created them are done
  • the variables need to be deallocated unless use smart_pointer
  • To put variable on heap
    • use pointer, and pointer should always be initialized with nullptr!
    • allocation happens when the new operation is used
    • pointer needs to be deleted after usage, it is also recommended to set it back to nullptr
      e.g.
      int * myIntegerPointer = nullptr; // create pointer
      myIntegerPointer = new int; // allocate memory
      *myIntegerPointer = 8; // assign value by dereference
      delete myIntegerPointer; // deallocate after done
      myIntegerPointer = nullptr; // set to nullptr to be safe

Pointer

  • Pointer does NOT always point to memoery in heap
    • can point to variable on stack or another pointer
      e.g.
      int i = 8;
      int * myIntegerPointer = &i; // points to the variable
  • Null pointer converts to false in boolean expression
  • To avoid invalid pointer, can evaluate the pointer first
    e.g.
    bool isValidSalary = (anEmployee && anEmployee->salary > 0); // anEmployee is a pointer

Dynamic Allocated Arrays

  • Array that lives on heap
  • Can be allocated same as variable, but with []
    e.g.
    int arraySize = 8;
    int* myVariableSizedArray = new int[arraySize]; // allocate memory
     
    myVariableSizedArray[3] = 2; // assign value
     
    delete[] myVariableSizedArray; // delete
    myVariableSizedArray = nullptr;
  • AVOID malloc and free from C, USE new and delete
  • always pair new with delete and new[] with delete[]

Smart pointer

  • Smart pointers automatically deallocate memory when the smart pointer object goes out of scope
  • std::unique_ptr is analogous to ordinary pointer
    e.g.
    auto anEmployee = make_unique<Employee>();
    // or
    unique_ptr<Employee> anEmployee(new Employee);
     
    // equivalent to
    Employee* anEmployee = new Employee;
    delete anEmployee;
  • std::shared_ptr allows pointer to shared
    e.g.
    auto anEmployee = make_shared<Employee>();
  • use std::unique_ptr by DEFAULT, and std::shared_ptr when needed

Const

  • Define constants
    e.g.
    const int versionNumberMajor = 2;
    const int versionNumberMinor = 1;
    const std::string productName = "Super Hyper Net Modulator"
  • Protect change to variables
    e.g.
    void mysteryFunction(const std::string* someString)
    {
      *someString = "Test"; // Will not compile.
    }

Pass by References

  • Avoid copying large class or structures
    e.g.
    void addOne(int i)
                    {
                    i++; // Has no real effect because this is a copy of the original
                    }
                    void addOne(int& i)
                    {
                    i++; // Actually changes the original variable
                    }
  • pass by const reference
    • avoid coppying
    • avoid changing variable

Type Inference

  • auto:
    • strips away reference and const, and makes a copy
      • to fix it, use auto & or const auto &
  • decltype
    • takes expression as argument and computes the type
      int x = 123;
      decltype(x) y = 456;

Class

Defining class

  • classes are usually defined in a header file (.h)
  • while their definitions usually are in a corresponding source file (.cpp).
  • It is recommended to make ALL data members private, and give access to them via public getters and setters
  • prefix data member with m
  • prefix constants with k
  • declare member functions that do not change any data members of an object being const
    • these functions are usually called "inspector"
      e.g.
      content_copy
      int getNumberOfMiles() const;

Initialize data members

  • constructor initializer
    e.g.
    content_copy 
    AirlineTicket::AirlineTicket()
      : mPassengerName("Unknown Passenger")
      , mNumberOfMiles(0)
      , mHasEliteSuperRewardsStatus(false)
    {}
  • initialize in body of constructor
    e.g.
    content_copy 
    AirlineTicket::AirlineTicket()
    {
      // Initialize data members
      mPassengerName = "Unknown Passenger";
      mNumberOfMiles = 0;
      mHasEliteSuperRewardsStatus = false;
    }

Stack vs Heap

  • Stack based classes
    e.g.
    content_copy 
    AirlineTicket myTicket; // Stack-based AirlineTicket
    myTicket.setPassengerName("Sherman T. Socketwrench");
  • Heap based classes
    e.g.
    content_copy 
    auto myTicket2 = make_unique<AirlineTicket>(); // Heap-based AirlineTicket
    myTicket2->setPassengerName("Laudimore M. Hallidue");

Initialization

Uniform initialization

  • post C++ 11, can initialize everything with {}
    e.g.
    content_copy 
    CircleStruct myCircle5{10, 10, 2.5};
    CircleClass myCircle6{10, 10, 2.5};
    int d{3}; // assign d to 3
  • Can be used to avoid narrowing
    e.g.
    content_copy 
    int a = {3.14}; // this will not compile
    int a = 3.14; // a = 3

Direct vs Copy initialization

  • post C++ 17,
    content_copy 
    // Copy list initialization
    auto a = {11}; // initializer_list<int>
    auto b = {11, 22}; // initializer_list<int>
    // Direct list initialization
    auto c {11}; // int
    auto d {11, 22}; // Error, too many elements.
